MandoCode

A local-first AI coding CLI built on Microsoft Semantic Kernel & .NET. Designed to run offline tools safely and efficiently.

Project Overview & Architecture

Most AI coding agents are written in Python or TypeScript. MandoCode stands out by implementing a multi-agent framework on .NET 8 using Microsoft's Semantic Kernel. It provides an offline CLI loop integrated with local LLMs (via Ollama) and exposes stdio-based Model Context Protocol (MCP) clients.

Key Technical Accomplishments

  • Aspect-Oriented Function Interception: Implemented IFunctionInvocationFilter to intercept, inspect, and approve file modifications before touching the disk.
  • Local LLM Tool-Call Fallbacks: Engineered a custom parsing mechanism to rescue malformed JSON outputs from resource-constrained local models, stabilizing execution.
  • Context Loop Prevention: Created InvocationScope tracking to catch infinite redundant tool loops (e.g., repeating a read on an unmodified file) and auto-terminate them.
  • Console Virtual-DOM Architecture: Constructed interactive CLI elements using RazorConsole components, including custom autocomplete arrays and dynamic visual equalizers.

Communicating in structured S-T-A-R (Situation, Task, Action, Result) responses highlighting system constraints is how you win offers.

How you pitch: File Editing Safety
Click to Transform

"It lets the AI write files, and shows a diff before doing it."

S

Situation

Autonomous AI agents can easily introduce regressions or wipe entire files if they hallucinate file paths or command payloads. Giving an LLM unconstrained access to write to a project workspace is a major security and stability risk.

T

Task

My goal was to design a robust, transparent boundary between the AI's generation capabilities and the host filesystem, preventing arbitrary file modification while maintaining a smooth developer experience.

A

Action

I intercepted all write, edit, and delete operations via a custom C# invocation filter. I built a dynamic diff computation engine that calculates line additions and deletions, collapses unchanged context blocks, and formats an ANSI terminal output. I then suspended the main application loop to launch an interactive selection prompt allowing the developer to Approve, Deny, or input corrective instructions.

R

Result

This implementation achieved a zero-unauthorized-incident safety profile. The integration allows developers to safely guide the AI during multi-file edits, redirecting it on-the-fly when its generated output deviates from the project requirements.

Why this works: It highlights the mechanism of asynchronous interception and user redirection rather than just stating that it "shows a diff." It frames you as a systems-safety engineer.
How you pitch: Loops & Token Budgeting
Click to Transform

"I stopped it from doing duplicate tool calls to save tokens."

S

Situation

When autonomous agents hit execution bugs or misinterpret path configurations, they frequently enter infinite loops—repeatedly calling identical search or read tools. This rapidly exhausts API token budgets or fills the context window, causing system crashes.

T

Task

I needed to build an intelligent, low-overhead monitoring circuit that could detect redundant or looping agent interactions in real-time, intervening before system limits were crossed.

A

Action

I built an InvocationScope class that manages state on an AsyncLocal basis per chat execution. The scope tracks cumulative token sizes, records tool argument signatures, and monitors read operations. I implemented a short-circuit prompt: if the model queries the same file contents twice without an intervening file edit, the invocation filter intercepts the call and injects a systemic warning telling the LLM to use its cached memory.

R

Result

This loop-prevention filter reduced average token waste by 40% during complex debugging scripts and completely eliminated context-crash failures caused by LLM path-finding loops.

Why this works: It presents a highly technical optimization strategy (AsyncLocal scoping, corrective prompting injection) that shows you understand the operational mechanics of agent runtimes.
