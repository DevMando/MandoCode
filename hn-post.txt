Body

Hey HN — I've been building MandoCode, an open-source AI coding assistant CLI that runs locally against Ollama (or its free cloud models). No API keys, no subscription. `dotnet tool install -g MandoCode`.

Most coding agents you see posted here are Python, Rust, or TypeScript. MandoCode is .NET 8 + Microsoft's Semantic Kernel for tool-calling, with the entire terminal UI built as Razor components running through RazorConsole (a virtual-DOM-over-Spectre.Console renderer). I wanted to see how far that stack could go for an agentic CLI, and the answer turned out to be: pretty far.

A few things I think are genuinely useful:

- Diff-gated writes. Every file write or delete is intercepted and shown as a colored diff before it touches disk — approve, deny, or redirect with new instructions. Per-file or session-wide bypass when you trust it.
- `@`-file references with autocomplete, `!shell` escape, `/` slash commands.
- Task planner that detects multi-step requests, generates a plan via a Semantic Kernel tool call, and gates execution behind approval.
- MCP client — drop any Claude-Desktop-shaped `mcpServers` block into the config and its tools light up alongside the built-ins. stdio + HTTP/SSE.
- Web search + fetch via DuckDuckGo, no key needed.
- A built-in lofi/synthwave player with a waveform visualizer in the corner. Yes, this is silly. I don't care.

One honest caveat: right now it works noticeably better with cloud models (Ollama's free `minimax-m2.7:cloud` is the default) than with small local ones. I've been testing on an RTX 3080, and the smaller qwen3.5 sizes hallucinate tool calls or miss multi-step plans more than I'd like. Bigger local models (9B+) hold up better, but I'm still figuring out the right combination of system prompt, fallback function parsing, and per-turn budget enforcement to get small-model reliability where I want it. If you've shipped agentic features that survive a 3B-parameter model, I'd love to hear how.

Free cloud models work out of the box once you `ollama signin`. Local works fully offline; the first-run wizard picks a model tier matched to your hardware (qwen3.5 0.8b → 9b).

Repo: https://github.com/DevMando/MandoCode

Happy to answer questions about the Semantic Kernel side, the diff-approval interception, or why anyone would put an MP3 player in a coding tool.
